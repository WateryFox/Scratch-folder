This repository contains my early work in visual programming using the Scratch platform developed by MIT. While these projects are built with blocks rather than text, they represent the foundation of my computational thinking. Working in Scratch allowed me to master the core logic of game development such as "broadcast" messaging, coordinate based movement, and variable management without the distraction of syntax errors. This archive serves as a record of my first dive into interactive storytelling and game mechanics.

Learning the Fundamentals
Even in these early projects, I was practicing complex programming concepts. I used "Forever" loops to handle constant game states and "If Then" blocks to create collision logic and player boundaries. One of the most important things I learned here was the concept of Concurrency how to make multiple "sprites" run their own code at the same time while communicating with each other through global variables or broadcast signals. These are the same principles I now use in Unity and Python, just translated into a different medium.

From Blocks to Architecture
Many of the mechanics I first explored in Scratch like health bars, score tracking, and simple gravity became the blueprints for my later projects in Unity 2D and 3D. By experimenting with the "Pen" tool and cloning functions, I learned how to manage multiple objects on screen and draw dynamic shapes based on math. This folder highlights my long term interest in technology and shows that I have been building, testing, and refining my logic skills since my very first school projects.
